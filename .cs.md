# Lox Interpreter

A Gleam implementation of a Lox language interpreter exposed as a CLI tool. Supports four commands: `tokenize`, `parse`, `evaluate`, and `run`, each taking a source file path.

---

## Entry Point (`main.gleam`)

Accepts CLI arguments via `argv`. Dispatches on the first argument:

| Command | Pipeline |
|---|---|
| `tokenize <file>` | read file → `process_tokenize` → `resolve_tokenize_result` |
| `parse <file>` | read file → `process_parse` → `resolve_parse_result` |
| `evaluate <file>` | read file → `process_evaluate` → `resolve_evaluation_result` |
| `run <file>` | read file → `process_run` → `resolve_run_result` |
| anything else | print usage to stderr, exit with code 1 |

Terminates via `erlang:halt` with the resolved exit code.

---

## Data Definitions (`data_def.gleam`)

### Tokens

`Token` is an ADT covering:
- Single-character operators: `Bang`, `Equal`, `LeftParen`, `RightParen`, `LeftBrace`, `RightBrace`, `Comma`, `Dot`, `Minus`, `Plus`, `Semicolon`, `Slash`, `Star`
- Two-character operators: `BangEqual`, `EqualEqual`, `LessEqual`, `GreaterEqual`, `Less`, `Greater`
- Literals with payloads: `String(value: String)`, `Number(lexeme: String, value: Float)`, `Identifier(name: String)`
- Keywords: `And`, `Class`, `Else`, `For`, `Fun`, `If`, `NilToken`, `Or`, `Print`, `Return`, `Super`, `This`, `TrueToken`, `FalseToken`, `Var`, `While`
- `Eof`

`TokenWithLine` wraps a `Token` with a `line: Int`.

### AST

`Expr`:
- `Literal(value: LiteralValue)`
- `Grouping(inner: Expr)`
- `Variable(name: String)`
- `Unary(op: UnaryOp, right: Expr)`
- `Binary(op: BinaryOp, left: Expr, right: Expr)`

`LiteralValue`: `TrueLiteral | FalseLiteral | NilLiteral | NumberLiteral(Float) | StringLiteral(String)`

`UnaryOp`: `NotOp | NegateOp`

`BinaryOp`: `MultiplyOp | DivideOp | AddOp | SubtractOp | GreaterOp | GreaterEqualOp | LessOp | LessEqualOp | EqualEqualOp | NotEqualOp`

`Statement`:
- `PrintStatement(line: Int, expression: Expr)`
- `ExpressionStatement(line: Int, expression: Expr)`
- `VarStatement(line: Int, name: String, initializer: Expr)`

### Error and Result Types

- `TokenizationError`: `UnrecognizedChar(line_number: Int, unexpected_char: String)` | `UnterminatedString(line_number: Int)`
- `TokenizationResult`: `tokens: List(Token)`, `errors: List(TokenizationError)` (errors are non-fatal; tokenization always completes)
- `TokenizationResultWithLines`: same but `tokens: List(TokenWithLine)`
- `ParseError`: `ParseErrorAtToken(token: Token, message: String)` | `ParseErrorAtEnd(message: String)`
- `LanguageError`: `TokenizationErrors(List(TokenizationError))` | `ParsingError(ParseError)`
- `RuntimeError`: `message: String`, `line: Int`
- `InterpretationResult`: `Completed(outputs: List(String))` | `Failed(outputs: List(String), error: RuntimeError)`
- `OperatorParseResult(op)`: `Operator(op, rest: List(Token))` | `NoOperator`

Exit codes: `exit_code_general_error = 1`

## Tokenizer (`tokenizer.gleam`)

`tokenize(source) -> TokenizationResult` — strips line info from `tokenize_with_lines`.

`tokenize_with_lines(source) -> TokenizationResultWithLines` — converts source to graphemes, then scans recursively.

### Scanning rules

- `\r\n`, `\n` — increment line counter, skip
- ` `, `\t` — skip
- `//` — skip until end of line (does not increment line)
- `"` — enter string scanning mode
- Two-character operators (`!=`, `==`, `<=`, `>=`) matched before single-character ones
- Single-character operators map directly to their token variants
- Digit character → scan number
- Alpha or `_` character → scan identifier/keyword
- Anything else → `UnrecognizedChar` error; continue scanning

**String scanning**: accumulates characters until closing `"`. Newlines inside strings increment the line counter and are included in the literal. An unterminated string (EOF without closing `"`) emits `UnterminatedString` and resumes normal scanning.

**Number scanning**: consumes all characters satisfying `is_number_char` (digits and `.`). Parses as `Float` (tries float parse, then int-to-float). On parse failure, emits `UnrecognizedChar` for the lexeme.

**Identifier/keyword scanning**: consumes alphanumeric + `_` characters. Matches against Lox keywords; anything else becomes `Identifier(lexeme)`.

**Keywords**: `and`, `class`, `else`, `false`, `for`, `fun`, `if`, `nil`, `or`, `print`, `return`, `super`, `this`, `true`, `var`, `while`.

The result always ends with an `Eof` token. Tokens and errors are accumulated in reverse then reversed once before returning.

## Parser

### Expression Parser (`parser/expression.gleam`, `parser.gleam`)

`parse(tokens: List(Token)) -> Result(Expr, ParseError)` — wraps tokens in default line 1, parses a single expression, and asserts no tokens remain beyond `Eof`.

`parse_with_rest(tokens: List(TokenWithLine)) -> Result(#(Expr, List(TokenWithLine)), ParseError)` — parses one expression and returns the remaining token list.

**Precedence (low to high)**:
1. Equality: `==`, `!=` — left-associative
2. Comparison: `>`, `>=`, `<`, `<=` — left-associative
3. Term: `+`, `-` — left-associative
4. Factor: `*`, `/` — left-associative
5. Unary: `!`, `-` — right-recursive
6. Primary

Left-associative levels are parsed via a shared `parse_left_associative` helper that parses the left operand, then loops consuming operator+operand pairs and folding into `Binary` nodes.

**Primary** matches: `true`, `false`, `nil`, number, string, identifier → `Variable(name)`, `(expr)` → `Grouping`. `Eof` or exhausted tokens → `ParseErrorAtEnd`. Any other token → `ParseErrorAtToken`.

Grouping requires a closing `)`, else `ParseErrorAtToken`/`ParseErrorAtEnd`.

### Program Parser (`parser/program.gleam`, `parser.gleam`)

`parse_program(tokens: List(TokenWithLine)) -> Result(List(Statement), ParseError)`

Parses a sequence of statements until `Eof` or empty token list.

**Statement dispatch**:
- `var` → var statement
- `print` → print statement
- anything else → expression statement

**Var statement**: expects `Identifier` after `var`. If followed by `;`, initializer defaults to `NilLiteral`. If followed by `=`, parses an expression then requires `;`. Other tokens → parse error.

**Print statement**: rejects an immediate `;` ("Expect expression."). Otherwise parses expression then requires `;`.

**Expression statement**: parses expression then requires `;`.

All statement parsers require a terminating `;` and produce appropriate error messages if it's missing.

## Evaluator

### Expression Evaluator (`evaluator/expression.gleam`, `evaluator.gleam`)

`evaluate(expr) -> Result(LiteralValue, String)` — evaluates in an empty environment.

`evaluate_in(expr, environment) -> Result(LiteralValue, String)` — environment is `Dict(String, LiteralValue)`.

**Rules**:
- `Literal(v)` → `Ok(v)`
- `Grouping(inner)` → recursively evaluate inner
- `Variable(name)` → look up in environment; missing → `Error("Undefined variable '<name>'.")`
- `Unary(NotOp, e)` → `bool_to_literal(!is_truthy(value))` for any value
- `Unary(NegateOp, e)` → negate if `NumberLiteral`, else error `"Operand must be a number."`
- `Binary(AddOp, ...)` → number + number, or string concatenation; mixed types → `"Operands must be two numbers or two strings."`
- Other binary numeric ops (`-`, `*`, `/`, `>`, `>=`, `<`, `<=`) → require both operands to be numbers, else `"Operands must be numbers."`. Comparison ops return `TrueLiteral`/`FalseLiteral`.
- `EqualEqualOp` / `NotEqualOp` → structural equality on `LiteralValue`, works across all types

**Truthiness**: `FalseLiteral` and `NilLiteral` are falsy; everything else is truthy.

**Number formatting** (`evaluator/format.gleam`): floats that end in `.0` are rendered without the decimal (e.g., `42.0` → `"42"`). Other floats use `float.to_string`.

### Program Interpreter (`evaluator/program.gleam`, `evaluator.gleam`)

`interpret(statements) -> InterpretationResult`

Maintains `ProgramState(outputs_rev, environment)`. Processes statements sequentially; stops and returns `Failed` on the first `RuntimeError`.

**Statement execution**:
- `PrintStatement` — evaluates expression, formats with `format.format`, appends to outputs
- `ExpressionStatement` — evaluates expression for side effects, discards value, state unchanged
- `VarStatement` — evaluates initializer, inserts `name → value` into environment

All evaluation errors are wrapped as `RuntimeError(message, line)` where `line` comes from the statement's stored line number.

`Completed` carries all printed outputs in order. `Failed` carries outputs printed before the error plus the `RuntimeError`.

## Command Runner (`command_runner.gleam`)

`execute_with_file(filename, process)` — reads the file with `simplifile`; on error, prints to stderr and returns exit code 1.

| Function | Steps |
|---|---|
| `process_tokenize(contents)` | `tokenize` |
| `process_parse(contents)` | `tokenize` (no-lines) → fail on tokenization errors → `parse` |
| `process_evaluate(contents)` | `tokenize` → `parse` → `evaluator.evaluate` |
| `process_run(contents)` | `tokenize_with_lines` → `parse_program` → `evaluator.interpret` |

For `process_parse` and `process_evaluate`, tokenization uses `tokenize` (no line info). For `process_run`, `tokenize_with_lines` is used to preserve line numbers for runtime errors.

Any tokenization errors abort the pipeline with `TokenizationErrors`. Any parse errors produce `ParsingError`.

## Command Result / Output (`command_result.gleam`)

Exit codes:
- `0` — success
- `65` — language error (tokenization or parse)
- `70` — runtime error

| Resolver | Output behavior |
|---|---|
| `resolve_tokenize_result` | Prints all tokens to stdout (via `tokenization_printer`), errors to stderr. Returns 65 if any errors, else 0. |
| `resolve_parse_result` | On success, prints AST via `ast_printer`. On error, prints language error to stderr and returns 65. |
| `resolve_evaluation_result` | On success, formats and prints the value to stdout. On runtime error, prints `"<message>\n\n[line 1]"` to stderr and returns 70. On language error, returns 65. |
| `resolve_run_result` | On `Completed`, prints each output line to stdout, returns 0. On `Failed`, prints prior outputs then `"<message>\n\n[line N]"` to stderr, returns 70. On language error, returns 65. |

## Printers

### Tokenization Printer (`tokenization_printer.gleam`)

Errors are printed first to stderr, then tokens to stdout.

Token format: `"<TYPE> <lexeme> <literal>"` where literal is `null` for non-value tokens, the string value for `STRING`, and the float representation for `NUMBER`. `EOF` has an empty lexeme.

Error format:
- `UnrecognizedChar`: `"[line N] Error: Unexpected character: <ch>"`
- `UnterminatedString`: `"[line N] Error: Unterminated string."`

### AST Printer (`ast_printer.gleam`)

S-expression format:
- `Literal` → value as string (`true`, `false`, `nil`, float string, or raw string value)
- `Grouping` → `"(group <inner>)"`
- `Variable` → identifier name
- `Unary` → `"(<op> <right>)"` where op is `!` or `-`
- `Binary` → `"(<op> <left> <right>)"` where op is the operator symbol

Number literals use `float.to_string` (preserves `.0`, unlike the evaluator formatter).

### Parse Error Printer (`parse_error_printer.gleam`)

- `ParseErrorAtToken(token, msg)` → `"[line 1] Error at '<lexeme>': <msg>"`
- `ParseErrorAtEnd(msg)` → `"[line 1] Error at end: <msg>"`

Line is hardcoded as `1` (expression-only parse mode has no line tracking).

## Utilities (`util.gleam`)

- `is_alpha(ch)` — true for `a-z`, `A-Z`, `_` (by codepoint range)
- `is_digit(ch)` — true for `0-9`
- `is_alpha_numeric(ch)` — `is_alpha || is_digit`
- `is_number_char(ch)` — `is_digit || ch == "."`

All functions operate on single-grapheme strings; multi-codepoint inputs return `False`.
